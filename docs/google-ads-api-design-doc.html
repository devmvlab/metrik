Note: this tool is externally accessible. Screenshots and mockups of the tool are included below.

Company Name: Metrik
Business Model: Metrik is a B2B SaaS platform that provides white-label performance dashboards for digital marketing agencies. Agencies subscribe to Metrik, customize the platform with their own branding, and use it to deliver real-time analytics dashboards to their end clients. Metrik is not an advertiser — it does not own any Google Ads accounts and does not create or manage campaigns. Instead, it acts as a reporting intermediary: each agency connects their clients' Google Ads accounts to Metrik via OAuth, and the platform displays that performance data within branded dashboards. Metrik serves multiple agencies simultaneously (multi-tenant), and each agency's data is fully isolated from others.
Tool Access/Use: The tool is accessed externally via a web application at the agency's custom domain (e.g., reports.agencyname.com or app.metrik.com.br/agency-slug). There are two user profiles:
  • Agency Admin — logs in to manage client accounts, connect integrations, and view all dashboards.
  • Client Viewer — receives an email invitation and logs in to view only their own performance dashboard.

The Agency Admin is the only user who initiates the Google Ads OAuth flow. They authorize Metrik to access a specific Google Ads customer account on behalf of their client. Once connected, both the admin and the invited client viewer can see the read-only performance data for that account within the dashboard. No user can modify campaigns, bids, budgets, or any other settings — the tool is strictly for reporting.
Tool Design:

1. OAuth Authorization Flow
When an Agency Admin connects a Google Ads account for a client, Metrik redirects the user to Google's OAuth 2.0 authorization endpoint with the scope https://www.googleapis.com/auth/adwords and access_type=offline to obtain a refresh token. After the user grants consent, Google redirects back to Metrik's callback route (/api/integrations/google-ads/callback), where the authorization code is exchanged for an access token and refresh token. Both tokens are immediately encrypted with AES-256 and stored in the database. The plain-text tokens are never persisted.

2. Customer ID Discovery
After the token exchange, Metrik calls customers:listAccessibleCustomers to retrieve the Customer ID associated with the authorized account. This ID is stored and used in all subsequent API queries.

3. Data Retrieval and Caching
When a user opens a dashboard, Metrik checks an internal cache table (integration_data_cache) for valid data (TTL: 4 hours). If no valid cache exists, it calls the Google Ads API using GAQL queries scoped to the stored Customer ID. Data is fetched only for the specific client whose dashboard is being viewed. Results are cached and served to the UI.

4. Display
The dashboard UI reads exclusively from the internal cache — it never calls the Google Ads API directly from the client side. Agencies can view account-level KPIs (cost, impressions, clicks, conversions, ROAS) and campaign-level breakdowns over selectable date ranges (last 7 days, last 30 days, custom range). No write operations are performed at any point.
API Services Called:
  • CustomerService — customers:listAccessibleCustomers
    Called once at OAuth callback time to discover the Customer ID linked to the authorized account.
  • GoogleAdsService — campaign resource (GAQL)
    Fetches campaign-level performance metrics: campaign.id, campaign.name, metrics.cost_micros, metrics.impressions, metrics.clicks, metrics.ctr, metrics.average_cpc, metrics.conversions, metrics.cost_per_conversion, metrics.all_conversions_value. Filtered by segments.date range and campaign.status = 'ENABLED'.
  • GoogleAdsService — customer resource (GAQL)
    Fetches account-level daily aggregated metrics: segments.date, metrics.cost_micros, metrics.impressions, metrics.clicks, metrics.conversions. Used to render time-series trend charts on the dashboard.
